JSON Format Examples

Sanfoundry Format (Quiz Platform)

Format from Sanfoundry quiz website with code blocks support

[
  {
    "number": 54,
    "question": "What will be the output of the following Python code snippet?",
    "options": {
      "a": "Error",
      "b": "True",
      "c": "False",
      "d": "No output"
    },
    "answer": "b",
    "explanation": "The code shown above is used to check whether a particular item is a part of a given set or not.",
    "code_blocks": [
      "z=set('abc$de')\nprint('a' in z)"
    ],
    "category": "Python Sets",
    "source_url": "https://www.sanfoundry.com/1000-python-questions-answers/"
  },
  {
    "number": 56,
    "question": "Which of the following is a feature of Python DocString?",
    "options": {},
    "answer": "d",
    "explanation": "Python has a nifty feature called documentation strings, usually referred to by its shorter name docstrings.",
    "code_blocks": [],
    "category": "Python Documentation",
    "source_url": "https://www.sanfoundry.com/1000-python-questions-answers/"
  }
]

Note: Options can be empty object {} for questions without multiple choice. Code blocks are joined with newlines.

Payal's Format (With Topics & Difficulty)

Extended format with topic, difficulty, and citation markers

[
  {
    "question": "What is the output of the following code?",
    "topic": "Python Basics",
    "difficulty": "Easy",
    "hint": "Check how range works with step.",
    "code": "print(len(list(range(1, 6, 2))))",
    "options": ["1", "2", "3", "Error"],
    "correct_answer": "3",
    "description": "range(1, 6, 2) yields 1, 3, 5; len is 3.",
    "reference": "https://docs.python.org/3/"
  }
]

Shubham's Format - Full Deck Object

Complete deck with metadata

{
  "name": "Python Basics",
  "description": "Fundamental Python concepts",
  "cards": [
    {
      "question": "What is a list in Python?",
      "hint": "It's a built-in data structure",
      "options": [
        "A key-value pair structure",
        "An ordered collection of items",
        "A function decorator",
        "A class attribute"
      ],
      "correct_answer": 1,
      "description": "A list is an ordered, mutable collection...",
      "reference": "https://docs.python.org/3/tutorial/datastructures.html",
      "code": "my_list = [1, 2, 3]\nprint(my_list[0])"
    }
  ]
}

Field Descriptions

  • question - The question text (required)
  • number - Question number (Sanfoundry format, optional)
  • topic - Topic/category (Payal's format only, optional)
  • category - Category (Sanfoundry format, optional)
  • difficulty - Difficulty level (Payal's format only, optional)
  • hint - A helpful hint (optional)
  • options - Answer choices:
    • Array format: ["opt1", "opt2", "opt3"] (Shubham/Payal)
    • Object format: {"a": "opt1", "b": "opt2"} (Sanfoundry)
    • Can be empty {} for non-multiple-choice questions
  • answer - Answer letter (a/b/c/d) for Sanfoundry format
  • correct_answer - For Shubham/Payal formats:
    • Integer index (0, 1, 2...) for Shubham's full format
    • Exact answer string for simple formats (e.g., "3", "continue")
  • description/explanation - Explanation shown after answering (optional)
  • reference/source_url - URL or reference link (optional)
  • code - Code snippet to display (optional)
  • code_blocks - Array of code snippets (Sanfoundry format, optional)

Supported Formats

  • Sanfoundry Format: Quiz format with options object (a, b, c, d) and answer field.
  • Payal's Format: Array with topic, difficulty, and citation markers (cleaned automatically).
  • Shubham's Simple Array: Just an array of card objects. Deck name comes from filename.
  • Shubham's Full Object: Object with "name", "description", and "cards" array.
  • correct_answer: Can be index number or exact string matching an option.
